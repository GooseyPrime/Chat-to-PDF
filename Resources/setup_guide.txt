# 🚀 ChatTranscriptConverter - Setup & Deployment Guide

## 📋 **Pre-Deployment Checklist**

### ✅ **Environment Setup**
1. Copy `.env.example` to `.env` and configure all required variables
2. Set up PostgreSQL database (local or cloud)
3. Configure Stripe account and webhook endpoints
4. Set up Firebase project for authentication
5. Install Node.js 18+ and npm/yarn

### ✅ **Infrastructure Requirements**
- **Database**: PostgreSQL 13+
- **Runtime**: Node.js 18+
- **Storage**: 10GB+ for PDF files
- **Memory**: 2GB+ RAM recommended
- **Chrome/Chromium**: Required for PDF generation

---

## 🔧 **Local Development Setup**

### **1. Clone & Install Dependencies**
```bash
cd C:\ChatTranscriptConverter\ChatTranscriptConverter
npm install
```

### **2. Database Setup**
```bash
# Install PostgreSQL locally or use cloud service (Neon, Supabase, etc.)
# Create database
createdb chattranscriptconverter

# Run migration
psql -d chattranscriptconverter -f database_migration.sql

# OR use Drizzle
npm run db:push
```

### **3. Configure Environment**
```bash
# Copy template
cp .env.example .env

# Edit .env with your values
# Minimum required:
DATABASE_URL=postgresql://username:password@localhost:5432/chattranscriptconverter
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
FIREBASE_PROJECT_ID=your-project-id
SESSION_SECRET=your-32-char-secret
```

### **4. Set Up Stripe**
1. **Create Products & Prices** in Stripe Dashboard:
   - Basic Weekly: $4.99/week
   - Pro Weekly: $9.99/week  
   - Pro Annual: $59.99/year

2. **Configure Webhook**:
   - URL: `https://yourdomain.com/api/stripe-webhook`
   - Events: `checkout.session.completed`, `payment_intent.payment_failed`, `customer.subscription.deleted`
   - Copy webhook secret to `STRIPE_WEBHOOK_SECRET`

### **5. Set Up Firebase**
1. Create Firebase project at https://console.firebase.google.com/
2. Enable Authentication with Email/Password
3. Generate service account key (optional, for server-side auth)
4. Configure environment variables

### **6. Start Development Server**
```bash
npm run dev
```

Access: http://localhost:5000

## 🚀 **Production Deployment**

### **Option A: Docker Deployment**
```dockerfile
# Dockerfile
FROM node:18-alpine

RUN apk add --no-cache \
    chromium \
    nss \
    freetype \
    harfbuzz \
    ca-certificates \
    ttf-freefont

ENV PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true \
    PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

EXPOSE 5000
CMD ["npm", "start"]
```

```bash
# Build and run
docker build -t chattranscriptconverter .
docker run -p 5000:5000 --env-file .env chattranscriptconverter
```

### **Option B: Cloud Platform Deployment**

#### **Vercel/Netlify**
```bash
# Build static files
npm run build

# Deploy
vercel --prod
# or
netlify deploy --prod
```

#### **Railway/Render**
1. Connect GitHub repository
2. Set environment variables in dashboard
3. Deploy automatically on push

#### **VPS/Dedicated Server**
```bash
# Install dependencies
sudo apt update
sudo apt install nodejs npm postgresql chromium-browser

# Clone repository
git clone <your-repo-url>
cd ChatTranscriptConverter

# Install & build
npm install
npm run build

# Set up process manager
npm install -g pm2
pm2 start dist/index.js --name chattranscriptconverter
pm2 startup
pm2 save

# Set up reverse proxy (nginx)
sudo nano /etc/nginx/sites-available/chattranscriptconverter
```

#### **Nginx Configuration**
```nginx
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 🔐 **Security Hardening**

### **Environment Security**
- Use strong, unique secrets for all environment variables
- Enable SSL/TLS in production (use Let's Encrypt)
- Implement rate limiting and DDoS protection
- Use environment-specific Firebase projects
- Restrict database access to application only

### **Application Security**
- Keep dependencies updated: `npm audit fix`
- Use security headers (already configured)
- Validate all user inputs
- Implement CSRF protection
- Use secure session configuration
- Regular security audits

### **Infrastructure Security**
- Keep server OS updated
- Use firewall rules (only allow ports 80, 443, 22)
- Disable root SSH access
- Use SSH keys instead of passwords
- Regular backup of database and files

## 🚨 **Troubleshooting**

### **Common Issues**

#### **1. PDF Generation Fails**
```bash
# Check Chrome installation
which chromium-browser
# or
which google-chrome

# Install missing dependencies
sudo apt install -y gconf-service libasound2 libatk1.0-0 libcairo2 libdrm2 libfontconfig1 libgconf-2-4 libgdk-pixbuf2.0-0 libgtk-3-0 libnspr4 libpango-1.0-0 libxss1 fonts-liberation
```

#### **2. Database Connection Issues**
```bash
# Test connection
psql -d $DATABASE_URL -c "SELECT 1;"

# Check connection string format
# postgresql://username:password@host:port/database?sslmode=require
```

#### **3. Stripe Webhook Issues**
```bash
# Check webhook logs in Stripe Dashboard
# Verify webhook secret matches environment variable
# Ensure endpoint is accessible from internet

# Test webhook locally with ngrok
ngrok http 5000
# Update webhook URL to ngrok URL
```

#### **4. Authentication Issues**
```bash
# Verify Firebase configuration
# Check if email verification is required
# Ensure Firebase rules allow authentication
```

### **Debug Mode**

# Enable debug logging
DEBUG=* npm run dev

# Check application logs
tail -f /var/log/chattranscriptconverter/app.log

# Monitor database queries

log_statement = 'all'

## 📈 **Scaling Considerations**

- Consider microservices architecture

### **Database Scaling**

- Connection pooling (PgBouncer)
- Database sharding for large datasets
- Regular performance optimization

### **File Storage Scaling**
- Move to cloud storage (AWS S3, Google Cloud Storage)
- Implement CDN for PDF downloads
- Use separate storage service/microservice
- Implement file cleanup automation
